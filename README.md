# AI Config Hub

这是一个用于统一管理多个 AI Agent（如 Claude Code, Cursor, Antigravity）配置、技能和规则的中心化仓库。

## 核心功能

*   **统一 Skill 管理**：将所有自定义 `SKILL.md` 集中存放，避免在不同 Agent 目录下重复安装。
*   **跨 Agent 同步**：一键将配置同步到 Claude (.claude.json), Cursor (.cursor/mcp.json) 和 Antigravity (.gemini/)。
*   **层级化规则**：区分全局通用规则 (Universal Rules) 与特定工程模板 (Project Templates)。
*   **开箱即用**：支持通过脚本快速初始化新环境。

## 目录结构

```text
.
├── skills/          # 自定义专家技能包 (如 linux-kernel-expert)
├── mcp/             # 全局 MCP 服务配置文件 (master_mcp.json)
├── rules/           # 全局通用行为准则 (如 universal-agent-rules.md)
├── templates/       # 特定工程类型的配置模板 (如 android-bsp)
├── repos/           # 托管第三方技能仓库 (如 antigravity-skills)
└── sync-ai.sh       # 核心同步脚本
```

## 快速开始

### 1. 克隆仓库
```bash
git clone https://github.com/Leo-Lyne/ai-config-hub.git ~/ai-config-hub
```

### 2. 同步配置
运行同步脚本，建立所有必要的符号链接：
```bash
chmod +x ~/ai-config-hub/sync-ai.sh
~/ai-config-hub/sync-ai.sh
```

### 3. 应用配置
重启你的 AI Agent（如重启 VS Code 或重新运行 `claude` CLI），配置即可生效。

## 贡献与定制

*   **添加新技能**：在 `skills/` 下创建文件夹并编写 `SKILL.md`。
*   **修改全局规则**：在 `rules/` 下修改对应的 Markdown 文件。
*   **更新后同步**：每次修改 Hub 中的内容后，请再次运行 `./sync-ai.sh`。
